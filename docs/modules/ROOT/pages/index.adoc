== What Is Arconia Migrations?

Arconia Migrations is a curated catalog of https://docs.openrewrite.org[OpenRewrite] recipes that automate version upgrades and refactoring across the Spring ecosystem, the Arconia Framework, common Java testing frameworks, and more. Each recipe handles the mechanical work of moving between versions: dependency bumps, deprecated API replacements, configuration property renames, and package relocations.

The recipes are designed to be composable. Higher-level upgrade recipes (such as `UpgradeSpringBoot_4_0`) chain together smaller, focused recipes (such as `UpgradeSpringFramework_7_0` and `UpgradeJUnit_6`) so that a single command can take a project across multiple major versions at once.

NOTE: Arconia Migrations is under active development and has not yet reached a GA (1.0.0) release. Recipe identifiers, options, and behavior may change between releases. Try it out and share feedback on https://github.com/arconia-io/arconia-migrations[GitHub].

== OpenRewrite

Arconia Migrations relies on https://docs.openrewrite.org[OpenRewrite], an open-source automated refactoring framework, and builds on top of the open-source recipes from the OpenRewrite catalog, released under the Apache 2.0 license. See the xref:philosophy.adoc[Philosophy] page for what Arconia Migrations focuses on and how it fits into the wider OpenRewrite ecosystem.

== Prerequisites

Arconia Migrations recipes run on JDK 21 or later. The project you're migrating can target any Java version supported by the OpenRewrite tooling you use to run them.

The https://docs.arconia.io/arconia-cli/latest/[Arconia CLI] is the simplest option: it bundles a compatible set of Arconia Migrations and OpenRewrite libraries, so you don't need to match versions yourself. If you use the https://docs.openrewrite.org/reference/gradle-plugin-configuration[OpenRewrite Gradle plugin] or https://docs.openrewrite.org/reference/rewrite-maven-plugin[Maven plugin] directly, see their documentation for the tooling's own requirements.

== Quick Start

You can run any Arconia Migrations recipe with the https://docs.arconia.io/arconia-cli/latest/[Arconia CLI]:

[source,shell]
----
arconia rewrite run --recipe-name io.arconia.rewrite.spring.boot4.UpgradeSpringBoot_4_0
----

TIP: Pass the https://docs.arconia.io/arconia-cli/latest/migration/rewrite-run/[`--dry-run`] option to preview the changes a recipe would make without applying them to your project.

TIP: To see all recipes available in your project, run https://docs.arconia.io/arconia-cli/latest/migration/rewrite-discover/[`arconia rewrite discover`]. The Arconia CLI automatically bundles every Arconia Migrations library as well as common open-source libraries from the OpenRewrite catalog, so no extra configuration is needed.

== Running Recipes

Every recipe in this project is a standard OpenRewrite recipe. That means you can run it with any OpenRewrite-compatible tool. The guides in this documentation show three of them — the https://docs.arconia.io/arconia-cli/latest/[Arconia CLI], the https://docs.openrewrite.org/reference/gradle-plugin-configuration[OpenRewrite Gradle plugin], and the https://docs.openrewrite.org/reference/rewrite-maven-plugin[OpenRewrite Maven plugin] — but they're not the only options.

The Arconia CLI is the recommended way to apply these recipes. It bundles all Arconia Migrations libraries out of the box, exposes convenience subcommands (for example, `arconia update spring-boot --to-version 4.0`), and gives the smoothest developer experience. For most upgrades, the CLI is enough.

The OpenRewrite Gradle and Maven plugins are the right choice when you need more control: customizing recipe options, integrating the migration into an existing build pipeline, or using an on-prem Maven repository instead of Maven Central. They take more configuration than the CLI, but they expose the full surface of the underlying OpenRewrite plugin.

These recipes automate the mechanical, repetitive parts of an upgrade, but they are best-effort: some changes still need manual follow-up, and a recipe leaves a property or API alone when it can't safely rewrite it. Always review the result before committing, and preview it first with `--dry-run`. After running an upgrade, build and test your project to confirm the migration is complete.
